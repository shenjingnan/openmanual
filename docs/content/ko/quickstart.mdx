# 빠른 시작

이 가이드는 5분 안에 완전한 문서 사이트를 만들 수 있도록 도와줍니다.

## 설치

```bash
npm install -g openmanual
```

또는 pnpm을 사용하세요:

```bash
pnpm add -g openmanual
```

## 프로젝트 생성

### 1. 프로젝트 디렉토리 생성

```bash
mkdir my-docs && cd my-docs
```

### 2. 설정 파일 생성

최소한의 `openmanual.json` 설정을 생성하세요:

```json
{
  "name": "My Docs"
}
```

### 3. 홈페이지 문서 생성

```bash
mkdir content
```

`content/index.mdx`를 생성하세요:

```mdx
---
title: 환영합니다
---

# Hello OpenManual

제 첫 번째 문서입니다.
```

### 4. 개발 서버 시작

```bash
openmanual dev
```

`http://localhost:3000`에 접속하면 문서 사이트를 확인할 수 있습니다.

## CLI 명령어

| 명령 | 설명 |
|------|------|
| `openmanual dev` | 개발 서버 시작 (기본 포트 3000) |
| `openmanual dev -p 8080` | 지정 포트로 시작 |
| `openmanual build` | 정적 사이트 빌드 |
| `openmanual preview` | 빌드 산출물 미리보기 (기본 포트 8080) |
| `openmanual preview -d ./out` | 지정 산출물 디렉토리 미리보기 |

## 더 많은 페이지 추가

새로운 MDX 파일을 생성하고 `sidebar` 설정에 등록하기만 하면 됩니다:

```json
{
  "name": "My Docs",
  "sidebar": [
    {
      "group": "문서",
      "pages": [
        { "slug": "index", "title": "홈" },
        { "slug": "guide", "title": "사용 가이드" }
      ]
    }
  ]
}
```

해당하는 파일 구조:

```
content/
├── index.mdx      # slug: index
└── guide.mdx      # slug: guide
```
