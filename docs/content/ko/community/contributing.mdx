# 기여 가이드

OpenManual 프로젝트에 관심을 가져주셔서 감사합니다! 다양한 방법으로 참여하실 수 있습니다.

## 행동 강령

본 프로젝트에 참여함으로써 기여자 규약을 준수하고 친절하고 존중하는 태도로 소통하는 데 동의하게 됩니다.

## 기여 방법

### 버그 보고

[GitHub Issues](https://github.com/shenjingnan/openmanual/issues)를 통해 문제를 보고해주세요. 다음을 포함해주세요:

1. **명확한 제목** — 문제에 대한 간단한 설명
2. **재현 단계** — 재현 방법에 대한 상세한 설명
3. **예상 동작** — 예상되는 동작
4. **실제 동작** — 실제로 발생한 동작
5. **환경 정보** — 운영 체제, Node.js 버전 등

### 새 기능 제안

Issue에 다음을 작성해주세요:

1. **기능 설명** — 추가하고 싶은 기능
2. **사용 시나리오** — 왜 이 기능이 필요한지
3. **대안** — 고려해본 다른 방법

### 코드 제출

#### 개발 환경 설정

```bash
# 저장소 클론
git clone https://github.com/shenjingnan/openmanual.git
cd openmanual

# 의존성 설치
pnpm install

# 개발 모드
pnpm run dev

# 프로젝트 빌드
pnpm run build

# 테스트 실행
pnpm run test

# 코드 검사
pnpm run check
```

#### 브랜치 전략

- `main` — 메인 브랜치, 안정성 유지
- `feature/*` — 새 기능 브랜치
- `fix/*` — 버그 수정 브랜치
- `docs/*` — 문서 업데이트 브랜치

#### 커밋 규칙

[Conventional Commits](https://www.conventionalcommits.org/)를 따르세요:

```
<type>(<scope>): <description>
```

유형: `feat`, `fix`, `docs`, `style`, `refactor`, `perf`, `test`, `chore`

#### Pull Request 절차

1. 저장소 Fork
2. 기능 브랜치 생성 (`git checkout -b feature/my-feature`)
3. 변경 사항 적용
4. 테스트 통과 확인 (`pnpm run test`)
5. 코드 검사 통과 확인 (`pnpm run check`)
6. 변경 사항 커밋 (`git commit -m "feat: add my feature"`)
7. 브랜치 푸시 (`git push origin feature/my-feature`)
8. Pull Request 생성

#### PR 확인 목록

- 모든 테스트 통과
- lint 검사 통과
- 타입 검사 통과
- 새 기능에 해당 테스트 포함
- 문서 업데이트 (필요한 경우)
- Commit 메시지 규격 준수

## 개발 명령어

| 명령어 | 설명 |
|------|------|
| `pnpm run dev` | 개발 모드 (watch) |
| `pnpm run build` | 프로젝트 빌드 |
| `pnpm run test` | 테스트 실행 |
| `pnpm run lint` | 코드 검사 |
| `pnpm run typecheck` | TypeScript 타입 검사 |
| `pnpm run check` | 전체 검사 |
