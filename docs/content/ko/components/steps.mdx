Steps 컴포넌트는 자동 번호가 매겨진 단계별 가이드를 만드는 데 사용되며, 작업 흐름, 설치 단계 등 순서가 있는 콘텐츠를 표시하는 데 적합합니다.

## 기본 사용법

`Steps`로 여러 `Step`을 감싸고, 각 `Step`이 하나의 단계가 됩니다:

```mdx
<Steps>
  <Step>종속성 설치</Step>
  <Step>구성 파일 생성</Step>
  <Step>개발 서버 시작</Step>
</Steps>
```

<Steps>
  <Step>종속성 설치</Step>
  <Step>구성 파일 생성</Step>
  <Step>개발 서버 시작</Step>
</Steps>

## 콘텐츠가 있는 단계

각 `Step` 내부에는 다양한 콘텐츠를 포함할 수 있습니다:

```mdx
<Steps>
  <Step>
    ### OpenManual 설치

    다음 명령어로 설치합니다:

    \```bash
    pnpm add openmanual
    \```
  </Step>
  <Step>
    ### 구성 파일 생성

    프로젝트 루트 디렉토리에 `openmanual.json`을 생성합니다.
  </Step>
</Steps>
```

<Steps>
  <Step>
    ### OpenManual 설치

    다음 명령어로 설치합니다:

    ```bash
    pnpm add openmanual
    ```
  </Step>
  <Step>
    ### 구성 파일 생성

    프로젝트 루트 디렉토리에 `openmanual.json`을 생성합니다.
  </Step>
  <Step>
    ### 개발 서버 시작

    `pnpm run docs:dev`를 실행하여 개발 서버를 시작하면 브라우저에서 문서를 미리볼 수 있습니다.
  </Step>
</Steps>

## 속성

### Steps

`Steps`는 컨테이너 컴포넌트로, 추가 속성이 없습니다. `Step` 하위 컴포넌트를 감싸는 데 사용됩니다.

### Step

| 속성 | 타입 | 필수 | 설명 |
|------|------|------|------|
| `children` | ReactNode | 예 | 단계 콘텐츠, 임의의 MDX 콘텐츠 지원 |

## 주의사항

- 단계 번호는 1부터 자동으로 증가합니다
- 각 `Step` 내부에서 완전한 Markdown/MDX 구문을 지원합니다
- 튜토리얼, 설치 가이드 등 순서대로 실행해야 하는 시나리오에 적합합니다
