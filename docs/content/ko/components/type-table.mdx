---

TypeTable 컴포넌트는 구조화된 테이블 형식으로 TypeScript 유형 정의를 표시하는 데 사용되며, API 문서에서 인터페이스, 유형 별칭 등을 표시하는 데 적합합니다.

## 기본 사용법

```mdx
<TypeTable
  type={{
    name: {
      type: 'string',
      description: '사용자 이름',
      required: true,
    },
    age: {
      type: 'number',
      description: '사용자 나이',
    },
    email: {
      type: 'string',
      description: '사용자 이메일 주소',
      default: 'user@example.com',
    },
  }}
/>
```

<TypeTable
  type={{
    name: {
      type: 'string',
      description: '사용자 이름',
      required: true,
    },
    age: {
      type: 'number',
      description: '사용자 나이',
    },
    email: {
      type: 'string',
      description: '사용자 이메일 주소',
      default: 'user@example.com',
    },
    role: {
      type: "'admin' | 'user' | 'guest'",
      description: '사용자 역할',
      default: 'user',
    },
  }}
/>

## 구성 예제

다음은 `openmanual.json`의 `theme` 구성에 대한 유형 정의입니다:

<TypeTable
  type={{
    primaryHue: {
      type: 'number',
      description: '테마 색상 값, 범위 0-360',
    },
    darkMode: {
      type: 'boolean',
      description: '다크 모드 활성화 여부',
      default: 'true',
    },
  }}
/>

## 속성

### TypeTable

| 속성 | 타입 | 필수 | 설명 |
|------|------|------|------|
| `type` | `Record<string, TypeField>` | 예 | 유형 필드 정의 객체 |

### TypeField

각 필드 정의는 다음 속성을 포함합니다:

| 속성 | 타입 | 필수 | 설명 |
|------|------|------|------|
| `type` | string | 예 | 필드 유형 설명 |
| `description` | string | 아니오 | 필드 설명 |
| `required` | boolean | 아니오 | 필수 여부 |
| `default` | string | 아니오 | 기본값 |

## 주의사항

- `type` 속성의 키 이름은 필드명이며, 값은 필드 정의 객체입니다
- `type` 필드의 콘텐츠는 자유 텍스트로, 임의의 유형 설명 문자열이 될 수 있습니다
- 구성 항목, API 파라미터, 컴포넌트 Props 등 유형 정보를 표시하는 데 적합합니다
