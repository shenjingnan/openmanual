Steps コンポーネントは、自動採番付きのステップガイドを作成するために使用されます。操作フロー、インストール手順など順序のあるコンテンツの表示に適しています。

## 基本的な使い方

`Steps` で複数の `Step` をラップし、各 `Step` が一つのステップとなります：

```mdx
<Steps>
  <Step>依存関係のインストール</Step>
  <Step>設定ファイルの作成</Step>
  <Step>開発サーバーの起動</Step>
</Steps>
```

<Steps>
  <Step>依存関係のインストール</Step>
  <Step>設定ファイルの作成</Step>
  <Step>開発サーバーの起動</Step>
</Steps>

## 内容付きステップ

各 `Step` 内には豊富なコンテンツを含めることができます：

```mdx
<Steps>
  <Step>
    ### OpenManual のインストール

    以下のコマンドでインストールします：

    \```bash
    pnpm add openmanual
    \```
  </Step>
  <Step>
    ### 設定ファイルの作成

    プロジェクトのルートディレクトリに `openmanual.json` を作成します。
  </Step>
</Steps>
```

<Steps>
  <Step>
    ### OpenManual のインストール

    以下のコマンドでインストールします：

    ```bash
    pnpm add openmanual
    ```
  </Step>
  <Step>
    ### 設定ファイルの作成

    プロジェクトのルートディレクトリに `openmanual.json` を作成します。
  </Step>
  <Step>
    ### 開発サーバーの起動

    `pnpm run docs:dev` を実行して開発サーバーを起動すると、ブラウザでドキュメントをプレビューできます。
  </Step>
</Steps>

## プロパティ

### Steps

`Steps` はコンテナコンポーネントで、追加のプロパティはありません。`Step` 子コンポーネントをラップするために使用されます。

### Step

| プロパティ | 型 | 必須 | 説明 |
|------|------|------|------|
| `children` | ReactNode | はい | ステップの内容。任意の MDX コンテンツに対応 |

## 注意事項

- ステップ番号は 1 から始まり自動的に増加します
- 各 `Step` 内では完全な Markdown/MDX 構文がサポートされています
- チュートリアル、インストールガイドなど、順序通りに実行が必要なシーンに適しています
